## BUG-001: GHAP strategy enum schema mismatch

### Summary
Fixed a schema mismatch bug where the JSON schema for `start_ghap` and `update_ghap` MCP tools defined incorrect strategy enum values, causing validation errors when clients used the schema-compliant values.

### Changes
- Updated strategy enum in `start_ghap` tool definition from outdated underscored values to correct hyphenated values matching server validation
- Updated strategy enum in `update_ghap` tool definition with the same fix
- Added regression test `test_bug_001_schema_validation_alignment()` to prevent future drift between JSON schema and server validation

### Root Cause
The JSON schema in `_get_all_tool_definitions()` was never updated when the strategy enum values were finalized during initial implementation. The schema had 6 underscored values (`hypothesis_testing`, etc.) while the server validated against 9 hyphenated values (`systematic-elimination`, etc.).

### Files Changed
- `src/learning_memory_server/server/tools/__init__.py` (lines 418, 461)
- `tests/server/tools/test_ghap.py` (new regression test)
