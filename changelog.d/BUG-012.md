## BUG-012: Fix index_codebase hanging on large directories

### Summary
Fixed `index_codebase` MCP tool hanging when indexing directories containing `.venv/`, `node_modules/`, or other dependency directories by adding default exclusion patterns.

### Changes
- Added default exclusion patterns to `index_codebase` in `src/learning_memory_server/server/tools/code.py`
- Exclusions include: `.venv/`, `venv/`, `node_modules/`, `.git/`, `__pycache__/`, `dist/`, `build/`, `target/`, `.pytest_cache/`, `.mypy_cache/`, `.ruff_cache/`, `htmlcov/`, `.coverage`, `*.egg-info/`
- Added regression tests in `tests/indexers/test_bug_012_exclusions.py`

### Root Cause
The MCP tool was not passing `exclude_patterns` to `index_directory()`, causing the indexer to traverse and process all files including tens of thousands of files in virtual environments and dependency directories.
