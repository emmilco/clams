## SPEC-002-15: MCP Tools for GHAP and Learning

### Summary
Implemented 11 MCP tools for GHAP tracking, learning/value formation, and semantic search. These tools enable Claude Code agents to track working state through structured GHAP entries, perform experience clustering, validate and store values, and search experiences semantically.

### Changes

#### New Tool Modules
- **Added `src/learning_memory_server/server/tools/errors.py`**: Custom exception types (MCPError, ValidationError, NotFoundError, InsufficientDataError)
- **Added `src/learning_memory_server/server/tools/enums.py`**: Domain/strategy/root cause enums with validation functions
- **Added `src/learning_memory_server/server/tools/ghap.py`**: 5 GHAP tools
  - `start_ghap`: Begin tracking new GHAP entry
  - `update_ghap`: Update current GHAP entry
  - `resolve_ghap`: Mark GHAP as resolved with retry logic
  - `get_active_ghap`: Get current active GHAP
  - `list_ghap_entries`: List resolved entries with filters
- **Added `src/learning_memory_server/server/tools/learning.py`**: 5 learning tools
  - `get_clusters`: Retrieve cluster information for an axis
  - `get_cluster_members`: Get experiences in a cluster
  - `validate_value`: Validate proposed value against cluster centroid
  - `store_value`: Store validated value statement
  - `list_values`: List stored values with optional filters
- **Added `src/learning_memory_server/server/tools/search.py`**: 1 search tool
  - `search_experiences`: Semantic search across experiences

#### Supporting Services
- **Added `src/learning_memory_server/observation/__init__.py`**: ObservationCollector and ObservationPersister for GHAP management
- **Added `src/learning_memory_server/clustering/__init__.py`**: ExperienceClusterer stub
- **Added `src/learning_memory_server/values/__init__.py`**: ValueStore stub
- **Added `src/learning_memory_server/search/__init__.py`**: Searcher stub

#### Tests
- **Added `tests/server/tools/test_errors.py`**: Tests for error types (5 tests)
- **Added `tests/server/tools/test_enums.py`**: Tests for enum validation (16 tests)
- **Added `tests/server/tools/test_ghap.py`**: Tests for GHAP tools (21 tests)
- **Added `tests/server/tools/test_learning.py`**: Tests for learning tools (15 tests)
- **Added `tests/server/tools/test_search.py`**: Tests for search tools (8 tests)

#### Tool Registration
- **Updated `src/learning_memory_server/server/tools/__init__.py`**: Register all 11 tools with service initialization

### Technical Details
- Decorator-based registration using `@server.call_tool()`
- Consistent error handling with standardized error responses
- Input validation for all parameters (enums, text lengths, formats)
- Exponential backoff retry logic (1s, 2s, 4s) for resolve_ghap persistence
- Full type hints with mypy --strict compliance
- Structured logging via structlog
