## BUG-067: MCP server registration uses wrong transport and non-existent subcommand

### Summary
Fixed the installer's MCP server registration to use SSE transport instead of command-based (stdio) transport, matching the actual server implementation.

### Changes
- Changed `merge_mcp_server()` to accept an SSE URL instead of a command list
- Changed `register_mcp_server()` to build SSE URL from `settings.server_host` and `settings.server_port`
- Removed unused `dev_mode` and `dev_directory` parameters from `register_mcp_server()`
- Updated `step_register_mcp()` caller in `steps.py`
- Updated all tests to verify SSE format (`{"type": "sse", "url": "..."}`)
- Added regression test asserting no command-based fields in output
