## BUG-084: Daemon uses sys.executable (global Python) instead of venv

### Summary
Fixed the daemon and session-start hook to detect and use the virtualenv Python
interpreter instead of blindly using `sys.executable`, which could point to a
global Python lacking CALM's dependencies.

### Changes
- Added `get_python_executable()` helper to `src/calm/server/daemon.py` that
  resolves the correct Python via VIRTUAL_ENV env var, pyvenv.cfg detection,
  or sys.executable fallback
- Updated `daemon.start_daemon()` to use the helper
- Updated `hooks/session_start.ensure_server_running()` to use the helper
- Added 11 regression tests covering all resolution strategies and verifying
  both call sites use the helper
