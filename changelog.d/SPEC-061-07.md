## SPEC-061-07: Daemon Lifecycle and Error Recovery

### Summary
Code audit confirmed daemon start/stop/restart/status, stale PID recovery after
kill -9, Qdrant-unavailable graceful degradation, SessionStart hook auto-start,
and server.log diagnostic setup all work correctly. Added 35 new tests covering
these critical paths.

### Changes
- Added `tests/calm/test_daemon_lifecycle.py` with 35 tests covering:
  - `get_server_pid()` edge cases (empty, invalid, whitespace)
  - `is_server_running()` for running, dead, and invalid PIDs
  - `stop_server()` SIGTERM/SIGKILL flow and stale PID cleanup
  - Stale PID file recovery (kill -9 scenario)
  - CLI `server start/stop/status/restart` commands
  - Log file setup and directory creation
  - Signal handler and PID cleanup registration in server main
  - SessionStart hook auto-start with timeout and error handling
  - Qdrant lazy connection (server starts without Qdrant)
  - Error handler catches Qdrant exceptions at tool invocation time
