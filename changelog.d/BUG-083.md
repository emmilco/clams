## BUG-083: Consistent error handling across all MCP tool categories

### Summary
Standardized error handling across all MCP tool categories to return
structured error dicts `{"error": {"type": "...", "message": "..."}}` instead
of raising exceptions that the server dispatcher catches generically.

### Changes
- Converted memory tools (store, retrieve, list, delete) from raising
  ValidationError/MCPError to returning structured error dicts
- Converted code tools (index, search, find_similar) to return error dicts
- Converted git tools (index_commits, search_commits, get_file_history,
  get_churn_hotspots, get_code_authors) to return error dicts
- Converted context tools (assemble_context) to return error dicts
- Converted session tools (should_check_in) to return error dicts
- Converted journal tools (store, list, get, mark_reflected) to return error dicts
- Removed MCPError import from ghap.py, converted persist failure to error dict
- Updated 16 test files to check for error dicts instead of raised exceptions
- Added 24 regression tests covering all tool categories
- Server-level exception handler kept as safety net for unexpected errors
