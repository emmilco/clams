# Bug Report: BUG-073

## Title
get_churn_hotspots uses wrong attribute names on ChurnRecord

## First Noticed On Commit
44b2489 (current HEAD of main, Feb 8 2026)

## Reproduction Steps
1. Call `mcp__calm__get_churn_hotspots` with days=30 on any repo with commit history
2. Server returns error: `'ChurnRecord' object has no attribute 'path'`

## Expected Behavior
The tool should return a list of files with highest change frequency.

## Actual Behavior
AttributeError: `'ChurnRecord' object has no attribute 'path'`

## Root Cause
In `src/calm/tools/git.py:311-320`, the `get_churn_hotspots` tool formats `ChurnRecord` objects using wrong attribute names:
- `h.path` — should be `h.file_path` (the dataclass field name in `src/calm/git/base.py:83`)
- `h.commit_count` — should be `h.change_count` (the dataclass field name in `src/calm/git/base.py:84`)

The tool was written with different attribute names than the dataclass defines. No test covers the formatting code path with real `ChurnRecord` objects.

## Fix Plan
1. Change `h.path` to `h.file_path` in `src/calm/tools/git.py:313`
2. Change `h.commit_count` to `h.change_count` in `src/calm/tools/git.py:314`
3. Add regression test that creates `ChurnRecord` objects and verifies the tool formats them correctly

### Acceptance Criteria
- `get_churn_hotspots` tool returns formatted results without errors
- Regression test verifies correct attribute access on ChurnRecord
